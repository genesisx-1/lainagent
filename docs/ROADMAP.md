## Next Up: Agent Daemons (v7)

The big missing piece — agents that **stay alive and work autonomously** even when you're not actively coding.

### Persistent Agent Workers
- **Daemon mode**: Toggle an agent ON and it runs continuously in the background
- Each agent gets a **work queue** — assign it tasks and it processes them one by one
- Agent asks "what should I do?" when its queue is empty (configurable)
- Visual indicator on the office: agent glows/pulses when in daemon mode
- **Start/Stop controls** per agent in the UI
- Daemon logs visible in a dedicated tab

### Scheduled Tasks
- Cron-like scheduler: "every 30 min, have Gemini check X"
- "Every morning, have Qwen generate a report from CRM data"
- "Every hour, scrape these 5 URLs and update contacts"
- Schedule editor in the UI with enable/disable toggles

### Agent Memory & Context
- Each agent maintains a context file (what it's been told, what it's learned)
- Agents can read previous conversation history before responding
- "Remember this" command — saves info to agent's memory file
- Context window management: summarize old conversations

### Smart Task Routing
- Agent reports back results, system decides next step
- Chain tasks: "Gemini research X, then Qwen analyze the results, then Codex generate code"
- Conditional routing: "If scrape finds emails, have Gemini research them"
- Priority queue with deadlines
